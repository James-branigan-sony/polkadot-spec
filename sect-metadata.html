12. Metadata

The runtime metadata structure contains all the information necessary on how to interact with the Polkadot runtime. Considering that Polkadot runtimes are upgradable and, therefore, any interfaces are subject to change, the metadata allows developers to structure any extrinsics or storage entries accordingly.

The metadata of a runtime is provided by a call to Metadata_metadata (Section C.5.1.) and is returned as a scale encoded (Section A.2.2.) binary blob. How to interpret and decode this data is described in this chapter.

12.1. Structure​

The Runtime Metadata is a data structure of the following format:

(M,vm,R,P,te,ve,E,tr){\left({M},{v}_{{m}},{R},{P},{t}_{{e}},{v}_{{e}},{E},{t}_{{r}}\right)}(M,vm​,R,P,te​,ve​,E,tr​)
R=(r0,…,rn){R}={\left({r}_{{0}},\ldots,{r}_{{n}}\right)}R=(r0​,…,rn​)
P=(p0,…,pn){P}={\left({p}_{{0}},\ldots,{p}_{{n}}\right)}P=(p0​,…,pn​)
E=(e0,…,en){E}={\left({e}_{{0}},\ldots,{e}_{{n}}\right)}E=(e0​,…,en​)

where

  • M{M}M are the first four constant bytes, spelling "meta" in ASCII.

  • vm{v}_{{m}}vm​ is an unsigned 8-bit integer indicating the format version of the metadata structure (currently the value of 14).

  • R{R}R is a sequence (Definition 202) of type definitions ri{r}_{{i}}ri​ (Definition 169).

  • P{P}P is a sequence (Definition 202) of pallet metadata pi{p}_{{i}}pi​ (Section 12.2.).

  • te{t}_{{e}}te​ is the type Id (Definition 170) of the extrinsics.

  • ve{v}_{{e}}ve​ is an unsigned 8-bit integer indicating the format version of the extrinsics (implying a possible breaking change).

  • E{E}E is a sequence (Definition 202) of extrinsics metadata ei{e}_{{i}}ei​ (Definition 180).

  • tr{t}_{{r}}tr​ is the type Id (Definition 170) of the runtime.

Definition 169. Runtime Registry Type Entry​

A registry entry contains information about a type in its portable form for serialization. The entry is a data structure of the following format:

ri=(idt,p,T,D,c){r}_{{i}}={\left(\text{id}_{{t}},{p},{T},{D},{c}\right)}ri​=(idt​,p,T,D,c)
T=(t0,…,tn){T}={\left({t}_{{0}},\ldots,{t}_{{n}}\right)}T=(t0​,…,tn​)
ti=(n,y){t}_{{i}}={\left({n},{y}\right)}ti​=(n,y)

where

  • idt\text{id}_{{t}}idt​ is a compact integer indicating the identifier of the type.

  • p{p}p is the path of the type, optional and based on the source file location. Encoded as a sequence (Definition 202) of strings.

  • T{T}T is a sequence (Definition 202) of generic parameters (empty for non-generic types).

    • n{n}n is the name string of the generic type parameter

    • y{y}y is a Option type containing a type Id (Definition 170).

  • D{D}D is the type definition (Definition 171).

  • c{c}c is the documentation as sequence (Definition 202) of strings.

Definition 170. Runtime Type Id​

The runtime type Id is a compact integer representing the index of the entry (Definition 169) in R,P{R},{P}R,P or E{E}E of the runtime metadata structure (Section 12.1.), depending on context (starting at 0{0}0).

Definition 172. Field​

A field of a data structure of the following format:

fi=(n,y,yn,C){{f}_{{i}}=}{\left({n},{y},{y}_{{n}},{C}\right)}fi​=(n,y,yn​,C)

where

  • n{n}n is an Option type containing the string that indicates the field name.

  • y{y}y is a type Id (Definition 170).

  • yn{y}_{{n}}yn​ is an Option type containing a string that indicates the name of the type as it appears in the source code.

  • C{C}C is a sequence of varying length containing strings of documentation.

Image 11. Metadata Type Fields​
Definition 173. Variant​

A struct variant of the following format:

vi=(n,F,k,C){v}_{{i}}={\left({n},{F},{k},{C}\right)}vi​=(n,F,k,C)

where

  • n{n}n is a string representing the name of the variant.

  • F{F}F is a possible empty array of varying length containing field (Definition 172) elements.

  • k{k}k is an unsigned 8-bit integer indicating the index of the variant.

  • C{C}C is a sequence of strings containing the documentation.

12.2. Pallet Metadata​

All the metadata about a pallet, part of the main structure (Section 12.1.) and of the following format:

pi=(n,S,a,e,C,e,i){p}_{{i}}={\left({n},{S},{a},{e},{C},{e},{i}\right)}pi​=(n,S,a,e,C,e,i)

where

  • n{n}n is a string representing the pallet name.

  • S{S}S is an Option type containing the pallet storage metadata (Definition 174).

  • a{a}a is an Option type (Definition 200) containing the type Id (Definition 170) of pallet calls.

  • e{e}e is an Option type (Definition 200) containing the type Id (Definition 170) of pallet events.

  • C{C}C is an Sequence (Definition 202) of all pallet constant metadata (Definition 179).

  • e{e}e is an Option type (Definition 200) containing the type Id (Definition 170) of the pallet error.

  • i{i}i is an unsigned 8-bit integer indicating the index of the pallet, which is used for encoding pallet events and calls.

Definition 174. Pallet Storage Metadata​

The metadata about pallets storage.

S=(p,E){S}={\left({p},{E}\right)}S=(p,E)
E=(e0,…,en){E}={\left({e}_{{0}},\ldots,{e}_{{n}}\right)}E=(e0​,…,en​)

where

  • p{p}p is the string representing the common prefix used by all storage entries.

  • E{E}E is an array of varying lengths containing elements of storage entries (Definition 175).

Definition 175. Storage Entry Metadata​

The metadata about a pallets storage entry.

ei=(n,m,y,d,C){e}_{{i}}={\left({n},{m},{y},{d},{C}\right)}ei​=(n,m,y,d,C)
C=(c0,…,cn){C}={\left({c}_{{0}},\ldots,{c}_{{n}}\right)}C=(c0​,…,cn​)

where

  • n{n}n is the string representing the variable name of the storage entry.

  • m{m}m is an enum type determining the storage entry modifier (Definition 176).

  • y{y}y is the type of the value stored in the entry (Definition 177).

  • d{d}d is a byte array containing the default value.

  • C{C}C is an array of varying lengths of strings containing the documentation.

Definition 180. Signed Extension Metadata​

The metadata about the additional, signed data required to execute an extrinsic.

ei=(n,y,a){e}_{{i}}={\left({n},{y},{a}\right)}ei​=(n,y,a)
